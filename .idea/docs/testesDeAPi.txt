Testes de API
Testar a comunicação e processamento entre
sistema que conversam via uma ou mais APIs

Quando não tem APIs, O teste se concentra:
- Telas/Páginas (UI): lendo, caro, complexo, Quebradiço
- Testes de Unidade: pode ser pouco obs exemplo é preciso testar como selecionasse cafe em produção

Os principais tipos de testes com APIs:
Saber testar o CRUD                  Nomes em API
- Incluir   - Create                 Post
- Consultar - Reach/Read/Research    Get
- Atualizar - Update                 Put
- Excluir   - Delete                 Delete

LGPD - Lei Geral de Proteção de Dados
- Dá o direito de ser esquecido
Excluir tem 2 tipos:
- Fisica: Apagar mesmo do banco de dados ---> Delete
- Lógica: "Desligar" o registro / cliente -> Update status

Além de testar o CRUD é importanta encadear transações, principalmente envolvendo
Tokens
